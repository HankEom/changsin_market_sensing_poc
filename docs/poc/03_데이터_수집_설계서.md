## 2. 크롤러 설계

### 2.1 공통 구조

```typescript
// 크롤러 인터페이스
interface CrawlerResult {
  source: string;
  url: string;
  title: string;
  content: string;
  published_at: string | null;
  image_url: string | null;
  language: 'en' | 'ko';
}

interface CrawlSummary {
  source: string;
  total: number;
  new_articles: number;
  duplicates: number;
  errors: number;
}
```

### 2.2 소스별 크롤링 전략

#### Packaging World

```
- 진입점: https://www.packworld.com/cosmetics
- RSS 피드 우선 확인 → 없으면 목록 페이지 HTML 파싱
- 선택자:
  - 기사 목록: article 또는 .article-card
  - 제목: h2 > a 또는 .article-title
  - 본문 링크: 제목 a[href]
  - 상세 페이지 본문: .article-body 또는 .field--body
- 수집 깊이: 목록 1페이지 (최신 20건)
- 주기: 1일 1회
```

#### Cosmetics Design

```
- 진입점: https://www.cosmeticsdesign.com/Article
- 카테고리 필터: Packaging 관련
- 선택자:
  - 기사 목록: .Teaser
  - 제목: .Teaser-title
  - 본문 링크: a[href]
  - 상세 본문: .Article-body
- 수집 깊이: 최신 15건
- 주기: 1일 1회
```

#### Beauty Packaging

```
- 진입점: https://www.beautypackaging.com/contents/
- 선택자:
  - 기사 목록: .article-summary
  - 제목: h3 > a
  - 본문 링크: h3 > a[href]
  - 상세 본문: .article-content
- 수집 깊이: 최신 10건
- 주기: 1일 1회
```

### 2.3 크롤링 실행 방식

| 방식 | 설명 | 용도 |
|------|------|------|
| **수동 트리거** | 대시보드 [수집 실행] 버튼 → POST /api/crawl | 시연, 테스트 |
| **Vercel Cron** | vercel.json에 cron 설정 (1일 1회) | 자동 수집 |

```json
// vercel.json
{
  "crons": [
    { "path": "/api/cron/crawl", "schedule": "0 6 * * *" }
  ]
}
```

## 3. 데이터 정제 파이프라인

### 3.1 정제 단계

```
원시 HTML
  → [1] HTML 태그 제거 (cheerio)
  → [2] 불필요한 요소 제거 (광고, 네비게이션, 사이드바)
  → [3] 텍스트 정규화 (연속 공백 제거, 줄바꿈 정리)
  → [4] 본문 길이 검증 (100자 미만 → 스킵)
  → [5] 언어 감지 (영어/한국어)
  → 정제된 텍스트
```

### 3.2 중복 제거

```
1차: URL 기반 완전 중복
  → Supabase에 동일 URL 존재 여부 확인
  → 존재 시 INSERT 스킵

2차: 제목 유사도 기반 근사 중복 (PoC에서는 1차만 적용)
```

## 4. 데이터 저장

### collected_articles 테이블 스키마

```sql
CREATE TABLE collected_articles (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  source VARCHAR(100) NOT NULL,         -- 수집 소스명
  url TEXT UNIQUE NOT NULL,             -- 원문 URL (중복 방지 키)
  title TEXT NOT NULL,                  -- 기사 제목
  content TEXT NOT NULL,                -- 정제된 본문
  image_url TEXT,                       -- 대표 이미지 URL
  language VARCHAR(10) DEFAULT 'en',    -- 언어 코드
  published_at TIMESTAMPTZ,             -- 원문 발행일
  analyzed BOOLEAN DEFAULT FALSE,       -- 분석 완료 여부
  created_at TIMESTAMPTZ DEFAULT NOW()  -- 수집 일시
);

CREATE INDEX idx_articles_analyzed ON collected_articles(analyzed);
CREATE INDEX idx_articles_source ON collected_articles(source);
CREATE INDEX idx_articles_created ON collected_articles(created_at DESC);
```

## 5. 에러 처리 및 로깅

| 에러 유형 | 처리 | 로깅 |
|----------|------|------|
| HTTP 4xx/5xx | 해당 소스 스킵, 다음 소스 진행 | console.error + crawl_logs 테이블 |
| 타임아웃 (10초) | 해당 소스 스킵 | 타임아웃 발생 소스·시각 기록 |
| 파싱 실패 | 해당 기사 스킵 | 실패 URL·에러 메시지 기록 |
| Supabase 오류 | 전체 수집 중단 → 에러 반환 | DB 에러 상세 기록 |

### crawl_logs 테이블

```sql
CREATE TABLE crawl_logs (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  source VARCHAR(100),
  status VARCHAR(20),              -- success / error / timeout
  articles_found INTEGER DEFAULT 0,
  articles_saved INTEGER DEFAULT 0,
  error_message TEXT,
  executed_at TIMESTAMPTZ DEFAULT NOW()
);
```
