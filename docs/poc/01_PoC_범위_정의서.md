## 1. PoC 목적

| 항목 | 내용 |
|------|------|
| **목적** | 창신 의사결정자에게 AI 기반 용기 디자인 트렌드 센싱의 실제 동작을 시연하여 본 프로젝트(CS-MSA 전체 구축) 수주를 확보한다 |
| **핵심 메시지** | "실제 시장 데이터를 크롤링하여 트렌드를 감지하고, 창신 용기와 자동 매칭하여 선제 제안까지 생성하는 과정을 라이브로 보여준다" |
| **시연 대상** | 창신 경영진 / 기획팀 / 영업팀 |

## 2. PoC 제약 조건

| 항목 | 조건 |
|------|------|
| 개발 기간 | **1주 (5영업일)** |
| 개발 인력 | **1인** 풀스택 |

## 3. In-Scope (PoC 포함 범위)

### 3.1 핵심 기능 — 3개

| # | 기능 | 설명 | 대응 Signal |
|---|------|------|------------|
| 1 | **트렌드 수집 & 키워드 지수** | 글로벌 패키징 미디어를 실제 크롤링하여 키워드를 추출하고 트렌드 지수를 생성한다 | Signal 01 |
| 2 | **트렌드-용기 자동 매칭** | 감지된 트렌드 키워드와 창신 용기 DB를 벡터 유사도로 매칭한다 | Signal 02 |
| 3 | **선제 제안서 자동 생성** | 매칭 결과를 기반으로 영업팀용 제안 자료를 LLM으로 생성한다 | Signal 02 |

### 3.2 데이터 수집 범위

| 소스 | 대상 | 수집 방식 |
|------|------|----------|
| 패키징 전문 미디어 | Packaging World, Cosmetics Design, Beauty Packaging 등 | 웹 크롤링 (RSS + HTML 파싱) |
| 어워드 사이트 | Pentawards, Dieline Awards, iF Design Award (Packaging) | 웹 크롤링 |
| 뉴스 | 화장품 용기/패키징 관련 구글 뉴스 | News API 또는 크롤링 |

### 3.3 대시보드 화면 — 4개

| 화면 | 설명 |
|------|------|
| 메인 대시보드 | 트렌드 지수 요약 + 최근 알림 |
| 트렌드 지수 상세 | 키워드별 언급량 차트 + Top 키워드 카드 |
| 매칭 결과 | 트렌드-용기 매칭 결과 카드 목록 |
| 제안서 생성 | 매칭 결과 선택 → LLM 제안서 생성 + 미리보기 |

### 3.4 알림

| 채널 | 범위 |
|------|------|
| 대시보드 내 알림 | 트렌드 급증 알림 표시 |

## 5. 용기 DB 전략

실제 창신 용기 카탈로그 데이터 확보 여부에 따라 2-track으로 준비한다.

| Track | 조건 | 대응 |
|-------|------|------|
| **Track A** | 실제 데이터 확보 | 실데이터 기반 매칭 시연 (임팩트 최대) |
| **Track B** | 데이터 미확보 | 화장품 용기 업계 일반 카탈로그 기반 가상 DB 구성 (30~50종) — 형태·소재·마감·규격 포함 |

**가상 DB 구성 기준 (Track B)**:

| 형태 | 소재 | 마감 | 규격 예시 |
|------|------|------|----------|
| 에어리스 펌프 | PP, PETG | 무광 도장, UV 코팅 | 15ml, 30ml, 50ml |
| 드롭퍼 | 유리, PETG | 유광, 프로스트 | 15ml, 30ml |
| 튜브 | PE, 바이오PE | 핫스탬핑, 실크인쇄 | 30ml, 50ml, 100ml |
| 컴팩트 | PP, ABS | 사출 무지, 메탈릭 | 10g, 15g |
| 크림자 | PP, 유리, 아크릴 | UV 코팅, 메탈라이징 | 30ml, 50ml, 100ml |
| 미스트/스프레이 | PP, PET | 투명, 컬러 | 50ml, 100ml, 150ml |
| 리필 용기 | PP, 알루미늄 | 아노다이징, 무광 | 30ml, 50ml |

## 6. 일정 계획 (5일)

| Day | 작업 | 산출물 |
|-----|------|--------|
| **Day 1** | 데이터 수집 파이프라인 구축 | 크롤러 3개 소스 동작, 수집 데이터 DB 적재 |
| **Day 2** | LLM 분석 파이프라인 구축 | 키워드 추출, 트렌드 지수 계산, 벡터 임베딩 생성 |
| **Day 3** | 용기 DB + 매칭 엔진 구축 | 용기 DB 구성, 벡터 매칭 동작, 제안서 생성 API |
| **Day 4** | 프론트엔드 대시보드 구현 | 4개 화면 구현, Vercel 배포 |
| **Day 5** | 통합 테스트 + 시연 준비 | E2E 테스트, 시연 시나리오 리허설, Slack 알림 연동 |

## 7. 시연 시나리오

### 메인 시나리오: "다음 시즌 트렌드 용기를 먼저 제안합니다"

```
[시연 흐름 — 약 10분]

1. [대시보드 오픈] 
   "CS-MSA가 24시간 글로벌 패키징 미디어를 감시하고 있습니다"
   → 메인 대시보드에 실시간 수집 현황 표시

2. [트렌드 지수 확인]
   "이번 주 가장 급증한 키워드는 '에어리스 펌프'입니다"
   → 키워드별 언급량 차트, Top 5 카드 확인
   → 전주 대비 72% 급증 하이라이트

3. [용기 매칭 결과]
   "이 트렌드에 맞는 창신 생산 가능 용기를 자동으로 찾았습니다"
   → 매칭 결과 카드 3건 표시 (유사도 점수 포함)

4. [제안서 자동 생성]
   "클릭 한 번으로 영업 제안서가 만들어집니다"
   → [제안서 생성] 버튼 클릭 → LLM이 실시간으로 제안서 작성
   → 트렌드 분석 + 추천 용기 + 예상 효과 포함된 제안서 미리보기

5. [Slack 알림 데모]
   "중요한 트렌드가 감지되면 즉시 알림이 갑니다"
   → Slack 채널에 실시간 알림 수신 시연

6. [클로징]
   "이것이 Signal 01-02 영역입니다. 
    본 프로젝트에서는 원자재·규제·경쟁사·클라이언트까지 
    5대 영역을 모두 감시합니다"
```

## 8. 성공 기준

| 기준 | 목표 |
|------|------|
| 실제 데이터 수집 | 3개 이상 소스에서 실시간 크롤링 데이터 시연 |
| 키워드 추출 정확도 | 전문가 판단 기준 추출 키워드의 70% 이상 적합 |
| 매칭 결과 유의미성 | 매칭된 용기 상위 3건 중 1건 이상 현업 판단 적합 |
| 제안서 품질 | 생성된 제안서가 영업팀이 수정 없이 참고할 수 있는 수준 |
| 대시보드 완성도 | 4개 화면 정상 동작, 시각적으로 프로페셔널한 인상 |
| 시연 안정성 | 시연 중 에러 없이 전체 플로우 완료 |
